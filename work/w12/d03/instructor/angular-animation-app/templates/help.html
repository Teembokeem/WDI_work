Basic AngularJS Animations


AngularJS Animation Basics

Animations in AngularJS are completely based on CSS classes and animations baked into CSS3.

There are two ways an animation can be triggered in Angular:

  1. CSS: If we define animations in our CSS, Angular will automatically add the necessary classes to trigger the animations on elements with directives such as ng-hide ng-show ng-repeat ng-include ng-switch ng-if form ng-view/ui-view.

  2. JS: the ngAnimation module contains a $animate service that provides power and flexiblilty and is designed to be used in controllers and custom directives

There are two types of CSS3 animations: transition & keyframe animations. Keyframes are usually best for more advanced animations.

In general, CSS3 animations control the speed of changes made to CSS property values instead of them changing instantly.

Setup the App

  1. Install the ngAnimate Module

    The ngAnimate module script must be loaded into index.html after Angular's main script:

          
      <head>
        ...
        <script src="https://code.angularjs.org/1.5.3/angular.min.js"></script>
        <script src="https://code.angularjs.org/1.5.3/angular-animate.min.js"></script>
        ...
      </head>

    Be sure the version numbers are the same for the two Angular scripts.

  2. Include the ngAnimate Module as a Dependency

    The ngAnimate module needs to be included inside the array when defining your app's main module:

          
        ...
        angular.module('app', ['ui.router', 'ngAnimate']);
        ...
  3. Create an animations.css:

    Since Angular's animations use several CSS3 classes, it would be a good idea to store that CSS in a separate file.

    Create it, call it what you want, and be sure to load it into index.html. Something like this:

          
        <head>
          ...
          <link rel="stylesheet" href="css/animation.css">
          ...
        </head>
